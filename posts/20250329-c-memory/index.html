C語言記憶體操作概念

2025-03-29

  • c
  • pointer
  • memory

本篇範例主要提供給已經會用 Python、JavaScript、Ruby、Java 等較高階語言，同時想要補充C語言記憶體操作概念的朋友。

指標變數

引用 (Reference)

在一般變數面前加一個 & 符號，這時取出的不是值 (value)，而是該值存放在記憶體中的位址 (address)。

 1#include <stdio.h>
 2
 3int main() {
 4    int x = 123;
 5    printf("value of x: %d\n", x);    // 123
 6    printf("address of x: %p\n", &x); // 0x7fff210a8b9c
 7    // &x -> reference
 8
 9    long int y = &x; // please ignore `long` for now
10    printf("value of y with decimal integer format: %d\n", y); // 554339228
11    printf("value of y with heximal address format: %p\n", y); // 0x7fff210a8b9c
12
13    return 0;
14}

value of x: 123
address of x: 0x7fff210a8b9c
value of y with decimal integer format: 554339228
value of y with heximal address format: 0x7fff210a8b9c

註：在 64 位元機器上，位址佔 64 位元，但 gcc 的 C int 只有 32 位元，所以這裡用 long。(這點看不懂沒關係)

—

引用與解引用 (Reference and dereference)

指標變數 (pointer variable) 裡面存的不是一般值，而是專門存記憶體位址。在它前面加上 * 符號，取出的是該變數裡「存放的記憶體位址」所「指向」的值。

例如 int *y;* 的意思，是宣告一個存放 int 類型值位址的變數。它是專門用來「存位址」，而不是「存數值」的變數。宣告之後，直接從 y 取出的就是位址 (address)，用 *y 取出的則是裡面位址指向的值 (value)。

 1#include <stdio.h>
 2
 3int main() {
 4    int x = 123;
 5    printf("value of x: %d\n", x);    // 123
 6    printf("address of x: %p\n", &x); // 0x7ffcaed1a40c
 7    // &x -> reference
 8
 9    int *y;
10    y = &x;
11    printf("value of y with decimal integer format: %d\n", y); // -1361992692
12    printf("value of y with heximal address format: %p\n", y); // 0x7ffcaed1a40c
13    printf("value at address pointed by y: %d\n", *y);         // 123
14    // *y -> dereference
15
16    return 0;
17}

value of x: 123
address of x: 0x7ffcaed1a40c
value of y with decimal integer format: -1361992692
value of y with heximal address format: 0x7ffcaed1a40c
value at address pointed by y: 123

—

這裡 y 指向了 x 存數值的地方，所以不管從 x修改還是從 *y 修改，都會影響另一邊。

 1#include <stdio.h>
 2
 3int main() {
 4    int x = 123;
 5    int *y = &x; // combine `int *y;` and `y = &x;`
 6    printf("value of x: %d\n", x);          // 123
 7    printf("value pointed by y: %d\n", *y); // 123
 8    printf("\n");
 9
10    x = x * 2;
11    printf("value of x: %d\n", x);          // 246
12    printf("value pointed by y: %d\n", *y); // 246
13    printf("\n");
14
15    *y = *y * 2;
16    printf("value of x: %d\n", x);          // 492
17    printf("value pointed by y: %d\n", *y); // 492
18
19    return 0;
20}

value of x: 123
value pointed by y: 123

value of x: 246
value pointed by y: 246

value of x: 492
value pointed by y: 492

在函數內修改變數值

傳值 (pass by value)

 1#include <stdio.h>
 2
 3void double_int(int x) {
 4    x = x * 2;
 5}
 6
 7int main() {
 8    int x = 123;
 9    printf("value of x: %d\n", x); // 123
10
11    double_int(x); // does not change x in main
12    printf("doubled value of x: %d\n", x); // 123
13
14    return 0;
15}

value of x: 123
doubled value of x: 123

怎麼改了沒有效？

—

傳址 (pass by address)

 1#include <stdio.h>
 2
 3void double_int(int *x) {
 4    *x = *x * 2;
 5}
 6
 7int main() {
 8    int x = 123;
 9    printf("value of x: %d\n", x); // 123
10
11    double_int(&x); // changes x in main
12    printf("doubled value of x: %d\n", x); // 246
13
14    return 0;
15}

value of x: 123
doubled value of x: 246

因為傳入位址，所以函數內的 x 「指向」的地方，正是外面 x 的值，因此能做修改。

如果傳址只是為了在函數內修改外面的變數，那我用 return 不就好了嗎？如果處理一個基本型態的變數，的確用 return 即可。但傳址更重要的，是「避免複製資料」，尤其在遇到大型物件、陣列、字典、連結串列、樹等較大的資料結構時。

這也是為什麼許多更高階卻不暴露指標的程式語言，在傳遞非基本型態 (例如 object, array, dict) 的變數「進出函式」時，會自動使用「避免複製資料」方式的原因。讓函式內外指向的，都是「同一份資料」。反過來，如果傳遞的是基本型態 (例如 bool, int, float, 甚至 string) 的變數進出函式時，就會「複製資料」，導致函式內的修改不會對外影響。

避免複製資料的方式有細分成 pass by address 與 pass by reference 等更多型態，有興趣的朋友可以再自行搜索，這裡不多談。

陣列與指標算術運算

變數 x 裡面存放的是什麼？就是指向陣列第一個元素的記憶體位址。

 1#include <stdio.h>
 2
 3int main() {
 4    int x[] = {1, 2, 3};
 5
 6    // print value of emements of array x
 7    printf("%d\n", x[0]); // 1
 8    printf("%d\n", x[1]); // 2
 9    printf("%d\n", x[2]); // 3
10    printf("\n");
11
12    // print address of emements of array x
13    printf("%p\n", &x[0]); // 0x7fff0c0b2f1c
14    printf("%p\n", &x[1]); // 0x7fff0c0b2f20
15    printf("%p\n", &x[2]); // 0x7fff0c0b2f24
16    printf("\n");
17
18    // print value of "variable x" and it's offets
19    // pointer arithmetic
20    printf("%p\n", x);     // 0x7fff0c0b2f1c
21    printf("%p\n", x + 1); // 0x7fff0c0b2f20
22    printf("%p\n", x + 2); // 0x7fff0c0b2f24
23    printf("\n");
24
25    // print value at addresses pointed by "variable x" and it's offets
26    // pointer arithmetic
27    printf("%d\n", *x);       // 1
28    printf("%d\n", *(x + 1)); // 2
29    printf("%d\n", *(x + 2)); // 3
30
31    return 0;
32}

陣列變數本身就是一個指標變數，它指向陣列第一個元素的記憶體位址。同理，要複製一個陣列需要對元素一個個做複製，而非直接用 = 指定給另一個陣列變數。

在函數內修改陣列元素

因為陣列變數存的，就是第一個元素的位址，所以傳入函數時仍可用 x[0], x[1]… 的格式操作。

 1#include <stdio.h>
 2
 3void print_int_array(int x[], int n) {
 4    printf("[");
 5    for (int i = 0; i < n; i++) {
 6        printf("%d", x[i]);
 7        if (i != n - 1) {
 8            printf(",");
 9        }
10    }
11    printf("]\n");
12}
13
14void double_int_array(int x[], int n) {
15    for (int i = 0; i < n; i++) {
16        x[i] = x[i] * 2;
17    }
18}
19
20int main() {
21    const int N = 3;
22
23    int x[] = {1, 2, 3};
24    print_int_array(x, N);
25
26    double_int_array(x, N);
27    print_int_array(x, N);
28
29    return 0;
30}

指標算術運算 (pointer arithmetic)

也可以把陣列變數的指標傳入函式，進行加減運算來指向陣列裡的不同元素，並修改元素的值。

 1#include <stdio.h>
 2
 3void print_int_array(int *x, int n) {
 4    printf("[");
 5    for (int i = 0; i < n; i++) {
 6        printf("%d", *(x + i)); // pointer arithmetic
 7        if (i != n - 1) {
 8            printf(",");
 9        }
10    }
11    printf("]\n");
12}
13
14void double_int_array(int *x, int n) {
15    for (int i = 0; i < n; i++) {
16        *(x + i) = *(x + i) * 2; // pointer arithmetic
17        // x[i] = x[i] * 2;
18    }
19}
20
21int main() {
22    const int N = 3;
23
24    int x[] = {1, 2, 3};
25    print_int_array(x, N);
26
27    double_int_array(x, N);
28    print_int_array(x, N);
29
30    return 0;
31}

Stack

以 factorial 為例

 1#include <stdio.h>
 2
 3int factorial(int n) {
 4    if (n == 0) {
 5        return 1;
 6    } else {
 7        return n * factorial(n - 1);
 8    }
 9}
10
11int main() {
12    int n = 3;
13    int result = factorial(n);
14    printf("factorial of %d: %d\n", n, result);
15
16    return 0;
17}

factorial of 3: 6

—

來看區域變數是怎麼被產生和收回的

 1// Pseudo code
 2main() {
 3    n = 3
 4    result = factorial(3) {
 5        n = 3
 6        return n * factorial(2) {
 7            n = 2
 8            return n * factorial(1) {
 9                n = 1
10                return n * factorial(0) {
11                    n = 0
12                    return 1
13                }
14            }
15        }
16    }
17    print(result)
18    return 0
19}

在 stack 裡的變數退出區塊時被收回，很單純。但如果你在底層區塊建立的變數資料，想要往上層傳遞，那個資料就不能放在 stack 裡，不然空間會被回收，上層無法取用資料。

Memory leak (注意，可能吃爆你電腦的記憶體)

 1#include <stdio.h>
 2
 3void do_your_task(int *x, int n) {
 4    for (int i = 0; i < n; i++) {
 5        x[i] = i * 2;
 6    }
 7}
 8
 9int *init_int_array(int size) {
10    int *arr = malloc(sizeof(int) * size);
11    return arr;
12}
13
14int main() {
15    int n = 30000;
16
17    for (int i = 0; i < n; i++) {
18        int *x = init_int_array(n);
19        do_your_task(x, n);
20
21        // Comment out free(x) to see memory leak on your computer wiht task manager.
22        // Remember to return memory space back to OS after you use.
23        free(x);
24    }
25
26    printf("press any key to exit...");
27    getchar();
28
29    return 0;
30}

程式碼錯綜複雜，開發者需要「確實追蹤」一段記憶體空間到什麼時候才不需要，不但非常花心力又容易遺忘。因此現代較高階的程式語言都有 Garbage Collection (GC) 功能，runtime 會自動追蹤並釋放不再使用的記憶體空間，但 GC 不是沒有代價。

Struct (可以對應為更高階語言的 Class)

傳 struct 值進入函式時，一樣修改不會在外生效

 1#include <stdio.h>
 2
 3typedef struct person_t {
 4    char* name;
 5    int age;
 6} person_t;
 7
 8void get_older(person_t person) {
 9    person.age++;
10}
11
12int main() {
13    person_t p1 = {
14        name : "Tom",
15        age : 30,
16    };
17
18    printf("Person %s is %d years old\n", p1.name, p1.age);
19
20    get_older(p1); // does not change p1 in main
21    printf("After 1 year...\n");
22    printf("Person %s is %d years old\n", p1.name, p1.age);
23
24    return 0;
25}

函式指標 (function pointer)

 1#include <stdio.h>
 2
 3int add(int a, int b) {
 4    return a + b;
 5}
 6
 7int multiply(int a, int b) {
 8    return a * b;
 9}
10
11int run_callback(int (*callback)(int a, int b), int a, int b) {
12    printf("Run the callback function %p...\n", callback);
13    return callback(a, b);
14}
15
16int main() {
17    int a = 3;
18    int b = 5;
19
20    printf("address of function add: %p\n", add);
21    printf("address of function multiply: %p\n", multiply);
22    printf("\n");
23
24    printf("add %d and %d is %d\n", a, b, run_callback(add, a, b));
25    printf("\n");
26
27    printf("multiply %d and %d is %d\n", a, b, run_callback(multiply, a, b));
28
29    return 0;
30}

address of function multiply: 0x5568d2f34181

Run the callback function 0x5568d2f34169...
add 3 and 5 is 8

Run the callback function 0x5568d2f34181...
multiply 3 and 5 is 15

Recap

  • 變數的引用與解引用 (Reference and dereference)。
  • 傳值與傳址 (pass by value and pass by address)。
  • 在函數內修改外層的變數，需要傳記憶體位址進去。
  • 在C語言裡，陣列變數存的就是第1個元素的記憶體位址。要存取後面的元素，就是以第1個元素的記憶體指標往後遞增即可。例如每個 int 為 4 byte (32-bit)，每往後一個元素的位址就是往後遞增4 (程式碼操作時 +1 即可，編譯器會依資料型態自動調整)。
  • Stack 資料與 heap 資料有不同特性。前者在宣告的執行區塊結束後就會被回收，因此不能分享給區塊外層的程式碼使用；後者需要開發者手動告知回收，可以傳記憶體位址分享到任意程式碼區塊。
  • Struct 和一般變數相同，可以決定傳值或傳址進函數。
  • 函數在C語言裡可以用記憶體位址定位，並可把位址傳入另一個函式內執行。在許多語言也是如此，也同樣可以把函數的記憶體位址印出來。
