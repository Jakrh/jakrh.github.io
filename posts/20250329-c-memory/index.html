記憶體操作概念：以 C 語言為例

本篇範例主要提供給已經掌握 Python、JavaScript、Ruby、Java 等較高階語言，但對程式在記憶體運作還不那麼瞭解的朋友。在瞭解記憶體指標、stack 與 heap 的運作概念後，要接觸更底層的程式語言會順暢很多。

指標變數

引用 (reference)

在一般變數面前加一個 & 符號 (address-of operator)，這時取出的不是值 (value)，而是該值存放在記憶體中的位址 (address)。

 1#include <stdio.h>
 2
 3int main() {
 4    int x = 123;
 5    printf("value of x: %d\n", x);    // 123
 6    printf("address of x with decimal integer format: %d\n", &x); // 631987892
 7    printf("address of x with heximal address format: %p\n", &x); // 0x7ffd25ab5eb4
 8    // &x -> reference
 9
10    return 0;
11}

value of x: 123
address of x with decimal integer format: 1897910100
address of x with heximal address format: 0x7ffe711fcf54

–

取出的記憶體位址，可以當作數值傳給另一個變數。

 1#include <stdio.h>
 2
 3int main() {
 4    int x = 123;
 5    printf("value of x: %d\n", x);    // 123
 6    printf("address of x with decimal integer format: %d\n", &x); // 554339228
 7    printf("address of x with heximal address format: %p\n", &x); // 0x7fff210a8b9c
 8    // &x -> reference
 9
10    // the address be can assigned to another vaiable
11    long int y = &x; // please ignore `long` for now
12    printf("value of y with decimal integer format: %d\n", y); // 554339228
13    printf("value of y with heximal address format: %p\n", y); // 0x7fff210a8b9c
14
15    return 0;
16}

address of x with heximal address format: 0x7fff210a8b9c

解引用 (dereference)

指標變數 (pointer variable) 裡面存的不是一般值，而是專門存記憶體位址。在它前面加上 * 符號 (indirection operator)，取出的是該變數裡「存放的記憶體位址」所「指向」的值。

例如 int *y; 的意思，是宣告一個存放 int 類型值位址的變數。它是專門用來「存位址」，而不是「存數值」的變數。宣告之後，直接從 y 取出的就是位址 (address)，用 *y 取出的則是裡面位址指向的值 (value)。

 1#include <stdio.h>
 2
 3int main() {
 4    int x = 123;
 5    printf("value of x: %d\n", x);    // 123
 6    printf("address of x with decimal integer format: %d\n", &x); // -1361992692
 7    printf("address of x with heximal address format: %p\n", &x); // 0x7ffcaed1a40c
 8    // &x -> reference
 9
10    int *y;
11    y = &x;
12    printf("value of y with decimal integer format: %d\n", y); // -1361992692
13    printf("value of y with heximal address format: %p\n", y); // 0x7ffcaed1a40c
14    printf("value at address pointed by y: %d\n", *y);         // 123
15    // *y -> dereference
16
17    return 0;
18}

value of x: 123
address of x with decimal integer format: -1361992692
address of x with heximal address format: 0x7ffcaed1a40c
value of y with decimal integer format: -1361992692
value of y with heximal address format: 0x7ffcaed1a40c
value at address pointed by y: 123

–

這裡 y 指向了 x 存數值的地方，所以不管從 x 修改還是從 *y 修改，都會影響另一邊。

 1#include <stdio.h>
 2
 3int main() {
 4    int x = 123;
 5    int *y = &x; // combine `int *y;` and `y = &x;`
 6    printf("value of x: %d\n", x);          // 123
 7    printf("value pointed by y: %d\n", *y); // 123
 8    printf("\n");
 9
10    x = x * 2;
11    printf("value of x: %d\n", x);          // 246
12    printf("value pointed by y: %d\n", *y); // 246
13    printf("\n");
14
15    *y = *y * 2;
16    printf("value of x: %d\n", x);          // 492
17    printf("value pointed by y: %d\n", *y); // 492
18
19    return 0;
20}

value of x: 123
value pointed by y: 123

value of x: 246
value pointed by y: 246

value of x: 492
value pointed by y: 492

在函數內修改變數的值

傳值 (pass by value)

 1#include <stdio.h>
 2
 3void double_int(int x) {
 4    x = x * 2;
 5}
 6
 7int main() {
 8    int x = 123;
 9    printf("value of x: %d\n", x); // 123
10
11    double_int(x); // does not change x in main
12    printf("doubled value of x: %d\n", x); // 123
13
14    return 0;
15}

value of x: 123
doubled value of x: 123

怎麼改了 x 沒有效？

–

傳址 (pass by address)

 1#include <stdio.h>
 2
 3void double_int(int *x) {
 4    *x = *x * 2;
 5}
 6
 7int main() {
 8    int x = 123;
 9    printf("value of x: %d\n", x); // 123
10
11    double_int(&x); // changes x in main
12    printf("doubled value of x: %d\n", x); // 246
13
14    return 0;
15}

value of x: 123
doubled value of x: 246

因為傳入位址，所以函數內的 x 「指向」的地方，正是外面 x 的值，因此能做修改。

雖然從人類的邏輯來看有分傳值與傳址，但從 C 語言的角度而言全都是傳值。

如果傳址只是為了在函數內修改外面的變數，那我用 return 不就好了嗎？如果處理一個基本型態的變數，的確用 return 即可。但傳址更重要的，是「避免複製資料」，尤其在遇到大型物件、陣列、字典、連結串列、樹等較大的資料結構時。

這也是為什麼許多更高階卻不暴露指標的程式語言，在傳遞非基本型態 (例如 object, array, dict) 的變數「進出函式」時，會自動使用「避免複製資料」方式的原因。讓函式內外指向的，都是「同一份資料」。反過來，如果傳遞基本型態 (例如 bool, int, float, 甚至 string) 的變數進出函式時，就會「複製資料」，導致函式內的修改不會對外影響。

另外其他語言的避免複製資料方式還有細分成 pass by address 與 pass by reference 等更多型態，有興趣的朋友可以再自行搜索，這裡不多談。

陣列與指標算術運算

陣列變數 x 裡面存放的是什麼？就是指向陣列第一個元素的記憶體位址。

 9    printf("%d\n", x[2]); // 3

陣列變數本身就是一個指標變數，它指向陣列第一個元素的記憶體位址。同理，要複製一個陣列需要對元素一個個做複製，而非直接用 = 指定給另一個陣列變數。

另外留意這三個記憶體位址，它們每個都間隔 4 bytes，也就是一個 int 的長度 (32-bit => 4-byte)。對指標做算術運算 +1，編譯器會自動轉換成該資料型態的長度並向後位移一個單位。如果換成對 long 型態的陣列變數 +1，它會往後位移 8 bytes，因為一個 long 的長度為 (64-bit => 8-byte) (在 x86-64 平台上)；如果 +2 則會往後位移 16 bytes，依此類推。

在函數內修改陣列元素

因為陣列變數存的，就是第一個元素的位址，所以傳入函數時仍可用 x[0], x[1]… 的格式操作。

 1#include <stdio.h>
 2
 3void print_int_array(int x[], int n) {
 4    for (int i = 0; i < n; i++) {
 5        printf("%d ", x[i]);
 6    }
 7    printf("\n");
 8}
 9
10void double_int_array(int x[], int n) {
11    for (int i = 0; i < n; i++) {
12        x[i] = x[i] * 2;
13    }
14}
15
16int main() {
17    const int N = 3;
18
19    int x[] = {1, 2, 3};
20    print_int_array(x, N); // 1 2 3
21
22    double_int_array(x, N);
23    print_int_array(x, N); // 2 4 6
24
25    return 0;
26}

指標算術運算 (pointer arithmetic)

也可以把陣列變數的指標傳入函式，進行加減運算來指向陣列裡的不同元素，並修改元素的值。

 1#include <stdio.h>
 2
 3void print_int_array(int *x, int n) {
 4    for (int i = 0; i < n; i++) {
 5        printf("%d ", *(x + i)); // pointer arithmetic
 6        // printf("%d ", x[i]);
 7    }
 8    printf("\n");
 9}
10
11void double_int_array(int *x, int n) {
12    for (int i = 0; i < n; i++) {
13        *(x + i) = *(x + i) * 2; // pointer arithmetic
14        // x[i] = x[i] * 2;
15    }
16}
17
18int main() {
19    const int N = 3;
20
21    int x[] = {1, 2, 3};
22    print_int_array(x, N); // 1 2 3
23
24    double_int_array(x, N);
25    print_int_array(x, N); // 2 4 6
26
27    return 0;
28}

記憶體分段的 stack 與 heap

一隻程式在編譯時，就會規劃好執行時的記憶體配置，把不同類型的資料劃分到不同的區段存放，並決定好各區段大小，將這些資訊寫入執行檔內。當作業系統載入這個執行檔，就會建立一個行程 (process)，也就是存活在記憶體裡面運作的程式實體。它接收作業系統提供的記憶體空間後，會按照事先規劃的區段進行記憶體管理。

常見的記憶體分段 (memory segmentation)

  • Text/Code Segment：儲存程式的機器碼。
  • Data Segment：儲存已初始化的全域變數。
  • BSS Segment：儲存未初始化的全域變數。
  • Heap Segment：儲存動態配置的資料 (如 malloc、new)。
  • Stack Segment：儲存函式呼叫堆疊 (call stack)、區域變數。

這裡專注在 stack 與 heap，如果好奇其他區段請自行查詢。

註：準確來說，C 語言程式碼的編譯實際上包含四個階段，分別為預處理、編譯、組譯和鏈結 (preprocessing, compilation, assembly, linking)，才會產生可執行檔。這非本文重點，故不贅述。

Stack 記憶體區段

區域變數的作用域

區域變數存放在 stack 區段，它們在被宣告的區塊執行結束後，就會自動消失，記憶體空間被釋放。例如 d 在最裡面的區塊執行完畢後，空間最先被釋放，接著才是 c，再來是 b，最後是 a，符合 stack 的後進先出 (Last In, First Out, LIFO) 特性。

所謂被釋放，只是被標記成「可用」而不再被保護，但並不會被自動填 0。所以資料有可能「殘留」，也有可能被其他操作給覆蓋掉。

 1#include <stdio.h>
 2
 3int main() {
 4    int a = 1;
 5    {
 6        int b = 2;
 7        {
 8            int c = 3;
 9            {
10                int d = 4;
11                printf("a: %d, b: %d, c: %d, d: %d\n", a, b, c, d);
12            }
13            printf("a: %d, b: %d, c: %d\n", a, b, c);
14        }
15        printf("a: %d, b: %d\n", a, b);
16    }
17    printf("a: %d\n", a);
18
19    return 0;
20}

在 stack 裡的變數退出區塊時被收回，很單純。但如果你在底層區塊建立的變數資料，想要往上層傳遞，那個資料就不能放在 stack 裡，不然空間會被釋放，無法確保上層取用到有效物件。

–

回傳區域變數位址

把 stack 物件位址往上層傳，無法確保上層拿到正確物件。因為它指向存放物件的記憶體空間，已經隨著原區塊執行結束一起被釋放掉了。上層有可能拿到已經被其他操作覆蓋，已經毀損的物件，也可能運氣好拿到完全殘留的物件。

而且這是個未定義行為 (undefined behavior)，也就是 C 語言規格書沒有明確定義的行為，各廠各家編譯器要怎麼實作都不算錯誤，可搜尋 function returns address of local variable。以我目前使用的 gcc v13.3.0 為例，它會回傳 NULL，導致外層拿到空指標 (更後面有說明空指標)，如果直接解引用就會引發程式崩潰 (crash)。

 2
 3void print_int_array(int x[], int n) {
 4    for (int i = 0; i < n; i++) {
 5        printf("%d ", x[i]);
 6    }
 7    printf("\n");
 8}
 9
10void double_int_array(int x[], int n) {
11    for (int i = 0; i < n; i++) {
12        x[i] = x[i] * 2;
13    }
14}
15
16int* init_int_array() {
17    int x[] = {1, 2, 3}; // in stack memory
18    return x;            // undefined behavior
19}
20
21int main() {

23
24    int* x = init_int_array();
25    print_int_array(x, N); // Crash: Segmentation fault (core dumped)

Heap 記憶體區段

用 malloc 函式申請存放在 heap 的記憶體空間。這樣的記憶體空間不會因為宣告時的程式碼區塊執行完畢而被自動釋放，因此可以傳遞記憶體位址分享到任意程式碼區塊。使用後需要手動告知釋放，不然會一直佔用空間直到整個程式結束 (因為編譯器不知道什麼時候能釋放)。

另外在釋放之後，要把該指標變數的內容設為 NULL，表示「不指向任何物件」，以避免懸空指標 (後面會講細節，這裡先意識到就好)。

 1#include <stdio.h>
 2#include <stdlib.h>
 3
 4void print_int_array(int *x, int n) {
 5    for (int i = 0; i < n; i++) {
 6        printf("%d ", x[i]);
 7    }
 8    printf("\n");
 9}
10
11void double_int_array(int *x, int n) {
12    for (int i = 0; i < n; i++) {
13        x[i] = x[i] * 2;
14    }
15}
16
17int *init_int_array(int size) {
18    int *arr = malloc(sizeof(int) * size); // in heap memory
19    if (arr == NULL) {
20        return NULL;  // malloc failed
21    }
22
23    arr[0] = 1;
24    arr[1] = 2;
25    arr[2] = 3;
26
27    return arr;
28}
29
30int main() {
31    int n = 3;
32
33    // allocating memory and initializing an array
34    int *x = init_int_array(n);
35    if (x == NULL) {
36        fprintf(stderr, "memory allocation failed\n");
37        return EXIT_FAILURE;  // exit with error (return non-zero)
38    }
39
40    print_int_array(x, n); // 1 2 3
41
42    double_int_array(x, n);
43    print_int_array(x, n); // 2 4 6
44
45    free(x);  // free heap memory
46    x = NULL; // avoid dangling pointer
47
48    return EXIT_SUCCESS;  // exit with success (return zero)
49}

C 語言程式的 stack 大小通常在編譯時就固定了，而 heap 的大小則可以在程式執行期間動態調整，程式能依需要向作業系統請求和釋放 heap 記憶體空間。

–

記憶體洩漏 (memory leak)

如果你的程式沒有重複不斷地申請 heap 記憶體空間，使用後不做釋放可能不會出什麼問題。反過來，如果你的程式需要重複不斷地申請 heap 空間，卻沒有在使用完後釋放回去，執行久了可能就會把電腦的記憶體空間耗盡，這現象稱為記憶體洩漏。在執行時間較長的程式容易突顯出來，例如網路服務、系統服務、遊戲、資料庫以及長期開著的 GUI 程式。

拿掉 free(x); 會生效。注意，可能吃爆你電腦的記憶體！

 1#include <stdio.h>
 2#include <stdlib.h>
 3
 4void do_your_task(int *x, int n) {
 5    for (int i = 0; i < n; i++) {
 6        x[i] = i * 2;
 7    }
 8}
 9
10int *init_int_array(int size) {
11    int *arr = malloc(sizeof(int) * size);
12    return arr;
13}
14
15int main() {
16    int n = 30000;
17
18    // Keep allocating memory for n times
19    for (int i = 0; i < n; i++) {
20        int *x = init_int_array(n);
21        if (x == NULL) {
22            fprintf(stderr, "memory allocation failed\n");
23            return EXIT_FAILURE;
24        }
25
26        do_your_task(x, n);
27
28        // Comment out free(x) to see memory leak on your computer with task manager.
29        // Remember: always return memory space back to OS after you use.
30        free(x);
31    }
32
33    printf("press any key to exit...");
34    getchar();
35
36    return EXIT_SUCCESS;
37}

實務上，因為程式碼錯綜複雜，開發者需要「確實追蹤」一段記憶體空間到什麼時候才不需要，不但非常花心力又容易遺忘。因此現代許多較高階的程式語言都提供垃圾回收 (garbage collection, GC) 機制，執行階段 (runtime) 會自動追蹤並釋放不再使用的記憶體空間，但 GC 也有額外的 CPU 與記憶體消耗等代價。

空指標 (null pointer)

不指向任何物件的「指標變數」，而且這個指標變數裡存放了「空指標常數」 (null pointer constant)。它通常是 ((void*)0) 或 0，表示「不指向任何物件」的特殊值，而不是真的指向記憶體位址 0，在C程式碼用 NULL 呈現。

因此我們在解引用 (dereference) 之前，可以先對它做檢查，例如 if (y != NULL)，確定不是空指標後才做解引用。反過來如果對空指標做解引用，往往會被作業系統的保護機制擋下來，程式被強制中斷而崩潰 (crash)。要進一步瞭解，可查詢 segmentation fault 與 access violation。

許多常見的語言也有類似錯誤，例如 Java 的 NullPointerException、C# 的 NullReferenceException、JavaScript 的 TypeError: Cannot read properties of null、Python 的 AttributeError: 'NoneType' object has no attribute '...'、以及 Ruby 的 NoMethodError: undefined method '...' for nil:NilClass。

 1#include <stdio.h>
 2
 3int main() {
 4    int *x = NULL;
 5    printf("value of x with decimal integer format: %d\n", x);
 6    printf("value of x with heximal address format: %p\n", x);
 7    printf("value at address pointed by x: %d\n", *x);
 8
 9    return 0;
10}

value of x with decimal integer format: 0
value of x with heximal address format: (nil)
Segmentation fault (core dumped)

什麼情況會遇到空指標？

操作檔案時，用來開啟檔案的 fopen 函數就會用到。如果執行成功，它會回傳檔案結構體 (FILE struct) 的記憶體位址，用來操作檔案。如果它執行失敗 (例如找不到檔案)，就會回傳 NULL 來表示不成功，使用前要做檢查。

 1#include <stdio.h>
 2#include <stdlib.h>
 3
 4int main() {
 5    // open a file for reading
 6    // return NULL if error opening file
 7    FILE *file = fopen("non_existent_file.txt", "r");
 8
 9    // check the pointer before dereferencing
10    if (file == NULL) {
11        fprintf(stderr, "error opening file\n");
12        return EXIT_FAILURE;  // exit with error (return non-zero)
13    }
14
15    // read file and print contents
16    char buffer[256];
17    while (fgets(buffer, sizeof(buffer), file) != NULL) {
18        printf("%s", buffer);
19    }
20
21    // close the file
22    fclose(file);
23
24    return EXIT_SUCCESS;  // exit with success (return zero)
25}

error opening file

懸空指標 (dangling pointer)

指標指向的物件「曾經」是有效的，但可能因為記憶體空間被自動回收或被手動釋放，現在已經無效，甚至被其他部份的程式使用中。如果對該位址的資料做修改，有可能導致程式出現非預期行為。

在 C 語言裡，不應該把會被自動回收 (stack 裡的) 的物件位址，傳給更外層使用。要給外層使用，就要在 heap 建立物件，才能往外層傳，並在使用後記得手動釋放。反過來，已經被手動釋放空間，要立即把對應的指標變數內容設為 NULL，避免後面誤以為指向有效物件，造成錯誤操作。

錯誤示範：把 stack 的資料傳給外層使用。

 1#include <stdio.h>
 2
 3int *get_value() {
 4    int x = 1;
 5    return &x;  // undefined behavior
 6}
 7
 8int main() {
 9    int *p = get_value();
10    printf("%d\n", *p);  // may crash
11
12    return 0;
13}

正確示範：要給外層用的資料建在 heap 上，使用後釋放，釋放後立即標示為空。

 1#include <stdio.h>
 2#include <stdlib.h>
 3
 4int *get_value() {
 5    int *x = malloc(sizeof(int));
 6    if (x == NULL) {
 7        return NULL;  // malloc failed
 8    }
 9
10    *x = 1;
11    return x;
12}
13
14int main() {
15    int *p = get_value();
16
17    // check before dereferencing
18    if (p == NULL) {
19        fprintf(stderr, "memory allocation failed\n");
20        return EXIT_FAILURE;
21    }
22
23    printf("%d\n", *p);
24
25    free(p);   // free the allocated memory
26    p = NULL;  // set pointer to NULL after freeing
27
28    return EXIT_SUCCESS;
29}

錯誤示範：釋放後沒有立即標示為空，之後的使用前檢查就會造成誤判。除了存取不該存取的地方，還引發重複釋放空間的錯誤。

 1#include <stdio.h>
 2#include <stdlib.h>
 3
 4int *get_value() {
 5    int *x = malloc(sizeof(int));
 6    if (x == NULL) {
 7        return NULL;  // malloc failed
 8    }
 9
10    *x = 1;
11    return x;
12}
13
14int main() {
15    int *p = get_value();
16
17    // check before dereferencing
18    if (p == NULL) {
19        fprintf(stderr, "memory allocation failed\n");
20        return EXIT_FAILURE;
21    }
22
23    printf("%d\n", *p);
24
25    free(p);  // free the allocated memory
26
27    // ... other code ...
28
29    // check if p is NULL before dereferencing
30    if (p == NULL) {
31        fprintf(stderr, "pointer is NULL\n");
32        return EXIT_FAILURE;
33    }
34
35    printf("%d\n", *p);  // mistakly dereferencing a freed pointer
36
37    free(p);  // double free error: undefined behavior
38
39    return EXIT_SUCCESS;
40}

野指標 (wild pointer)

對於區域變數與動態大小陣列，C 語言為了性能考量，不會主動初始化內容，所以裡面殘留了什麼資料不得而知，我們稱為垃圾值 (garbage value)。如果我們宣告了一個指標變數，但沒有設定它的內容 (初始化)，無法保證它會指向哪裡。當我們修改它指向位址的資料時 (例如 *y = 4;) ，可能導致程式出現非預期行為，C 語言就是這麼地自由。所以最好在宣告時就輸入 NULL，後面才容易判斷它是否指向有效的內容。

下面這個程式每次執行，都有可能輸出不同結果，甚至程式崩潰。

 1#include <stdio.h>
 2
 3int main() {
 4    int x;
 5    printf("uninitialized value of x: %d\n", x);
 6
 7    int *y; // wild pointer
 8    printf("uninitialized value of y with heximal address format: %p\n", y);
 9    printf("uninitialized value at address pointed by y: %d\n", *y);
10
11    return 0;
12}

uninitialized value of x: 32767
uninitialized value of y with heximal address format: 0x7fff6742bf68
uninitialized value at address pointed by y: 1732436337

 1#include <stdio.h>

 3int add(int a, int b) {
 4    return a + b;
 5}
 6
 7int multiply(int a, int b) {
 8    return a * b;
 9}

結構體 (struct) 傳遞

C 語言的結構體類似於更高階語言的類別 (class)，可定義各種欄位。與基本型別一樣，直接實例化的結構體是放在 stack 裡，這點反而和許多語言不同。傳遞結構體進入函式時，函式內的修改不會在外層生效。

 1#include <stdio.h>
 2
 3typedef struct person_t {
 4    char* name;
 5    int age;
 6} person_t;
 7
 8void get_older(person_t person) {
 9    person.age++;
10}
11
12int main() {
13    person_t p1 = {
14        name : "Tom",
15        age : 30,
16    };
17
18    printf("Person %s is %d years old\n", p1.name, p1.age);
19
20    get_older(p1); // does not change p1 in main
21    printf("After 1 year...\n");
22    printf("Person %s is %d years old\n", p1.name, p1.age);
23
24    return 0;
25}

Person Tom is 30 years old
After 1 year...
Person Tom is 30 years old

–

需要傳結構體的位址進去函式，修改時外層才會生效。

 1#include <stdio.h>
 2
 3typedef struct person_t {
 4    char* name;
 5    int age;
 6} person_t;

 8void get_older(person_t* person) {
 9    person->age++;
10    // (*person).age;

 2#include <stdlib.h>
 3
 4typedef struct person_t {

回顧

  • 變數的引用與解引用 (reference and dereference)。
  • 傳值與傳址 (pass by value and pass by address)，雖然從人類的邏輯來看有區分，但從 C 語言的角度而言全都是傳值。
  • 在函數內修改外層的變數，需要傳記憶體位址進去。
  • 在 C 語言裡，陣列變數存的就是第1個元素的記憶體位址。要存取後面的元素，就是以第1個元素的記憶體指標往後遞增即可。
  • Stack 資料與 heap 資料有不同特性。前者在宣告的執行區塊結束後就會被回收，因此不能分享給區塊外層的程式碼使用；後者需要開發者手動告知回收，可以傳遞記憶體位址分享到任意程式碼區塊。
  • 手動管理記憶體複雜且容易出錯，需要耗費開發者更多時間心力，所以許多程式語言提供垃圾回收機制。
  • 空指標 (null pointer) 表示不指向任何物件的「指標變數」，在使用前要做檢查。
  • 懸空指標 (dangling pointer)，其記憶體位址指向的物件已經無效了。
  • 野指標 (wild pointer)，未經初始化的指標變數，不知道它指向哪裡。
  • 函數在 C 語言裡可以用記憶體位址定位，並可把位址傳入另一個函式內執行，和許多語言相同。
  • 結構體 (struct) 的記憶體特性和基本型別相同，因此開發者可依需要，自由決定要放在 stack 還是 heap 裡。
