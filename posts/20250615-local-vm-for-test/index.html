筆記：安裝與設定本機測試用 Ubuntu 虛擬機

紀錄個人的本機測試用虛擬機安裝與設定，也供大家參考。這是反覆使用後，訂出我自己舒適的設置，做完想要的測試後可以用快照迅速回到原本狀態。

2025-06-15

    環境

    • M4 MacBook Air 13 - 24G RAM, 512G SSD
    • VirtualBox 7.1.10
    • Ubuntu Server 24.04.2

    下載 OS image

    • https://ubuntu.com/download/server

    設定 VirtualBox 虛擬機

    • 選專家模式
    • 在 “檔案 > 工具 > 網路 >「僅限主機」網路” 裡，新增一個 HostNetwork
    • 新增一台 Ubuntu VM，名為 Ubuntu Server
      • 先不指定安裝 ISO 映像檔 (不會啟用無人值守安裝)
      • RAM: 6144 MB
      • CPU Cores: 4
      • Storage: 160 GB
    • 修改 VM 網卡
      • 第一張網卡從 NAT 改成 HostNetwork
      • 啟用第二張網卡，並使用 NAT
    • 先擷取一個快照，命名為 Empty

    安裝 OS

    • 指定 VM 的 ISO 映像檔為剛下載的 Ubuntu Server ISO 檔
    • VM 開機

    安裝程式介面

    • 安裝的語言相關配置都選 English
    • 可以把 LVM 拿掉
    • 記得裝 openssh-server

    加入實體機 SSH 公鑰

    1ssh-copy-id YOUR_USER@YOUR_VM_IP

    Note: 如果實體機還沒有設 SSH key，可用 ssh-keygen -t ed25519 -C "your_email@example.com" 建立。

    用 SSH 登入 VM

    設定 sudo 免密碼

    1echo "${USER}   ALL=(ALL:ALL) NOPASSWD:ALL" | sudo tee "/etc/sudoers.d/nopasswd-${USER}"
2sudo chmod 440 "/etc/sudoers.d/nopasswd-${USER}"

    擷取 OS 安裝後的快照

    • 如果有點潔癖，可以先打 history -c
    • 將 VM 關機 (可以用 VirtualBox 的 ACIP 關機)
    • 擷取快照，命名為 Ubuntu 24.04.2 (如果 Ubuntu 出了新的 patch 版就用新版號)

    安裝 Docker

    直接用官方安裝流程

    https://docs.docker.com/engine/install/ubuntu/

    設定免 sudo 使用 Docker

    1sudo usermod -aG docker $USER

    擷取 Docker 安裝後的快照

    • 如果有點潔癖，可以先打 history -c
    • 將 VM 關機 (可以用 VirtualBox 的 ACIP 關機)
    • 擷取快照，命名為 Docker v28.2.2 (如果 Docker 出了新版，就用新版號)
